# ADR-0000: ADR Template

**Status:** Decided

## Context
- Need a consistent format for recording architecture decisions
- Team must understand rationale, alternatives, and trade-offs for each choice

## Decision
- Use MADR (Markdown Architectural Decision Records) format
- Required sections: Status, Context, Decision, Consequences
- Include a visualization when it clarifies the decision's impact
- Rejected **Nygard format** (lacks alternatives section) and **Y-Statement** (too terse)

## Consequences
- **Positive:** Consistent, reviewable, version-controlled decision history
- **Negative:** Overhead of maintaining records for every decision
